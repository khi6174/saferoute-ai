# SafeRoute AI 데모 스크립트

## 1. 문제 제기

택배 현장은 빠른 배송만으로 설명되지 않습니다. 같은 배송량이라도 비, 야간, 언덕, 좁은 골목, 주정차 난이도가 겹치면 기사에게 사고 위험이 크게 증가합니다.

SafeRoute AI는 기사 감시 시스템이 아닙니다. 기사 안전을 지원하고, 관리자가 물량과 경로를 균형 있게 조정하도록 돕는 AI 의사결정 지원 시스템입니다.

## 2. 핵심 메시지

> 우리는 배송을 더 빠르게 만드는 것이 아니라, 더 안전하게 만드는 AI를 만듭니다.

## 3. 3분 시연 흐름

1. `관악구 우천 고위험 시나리오`를 선택합니다.
2. 좌측 기사 위험도 현황에서 `지원 기사 A`가 고위험 상태임을 확인합니다.
3. 상세 패널에서 남은 배송, 누적 근무 시간, 현재 날씨를 확인합니다.
4. 위험 원인 TOP 요소에서 남은 배송, 근무 시간, 피로, 우천, 언덕/빌라 밀집 요인을 설명합니다.
5. `최단 경로`와 `안전 경로`를 비교합니다.
6. 시간이 조금 더 걸리더라도 위험도가 낮은 안전 경로를 AI가 추천한다는 점을 강조합니다.
7. 배송 스케줄 재조정 추천에서 물량 분산, 휴식, 배송 순서 변경, 저우선순위 배송 지연을 확인합니다.
8. `AI 추천 적용` 버튼을 누릅니다.
9. 관리자 대시보드와 일일 안전 리포트에서 조정 전/후 위험도 개선 효과를 확인합니다.

## 4. 발표 문장 예시

“SafeRoute AI는 배송 속도를 최적화하는 시스템이 아닙니다. 현재 배송 상황에서 어떤 기사에게 안전 지원이 필요한지 예측하고, 왜 위험한지 설명하며, 경로와 스케줄을 어떻게 바꾸면 위험을 낮출 수 있는지 제안합니다.”

“이 데이터는 실제 개인정보가 아니라 시뮬레이션 데이터입니다. 하지만 배송량, 피로, 날씨, 지역 특성, 경로 위험을 구조화했기 때문에 실제 서비스로 확장 가능한 의사결정 흐름을 보여줄 수 있습니다.”

## 5. 주의 문구

- 이 MVP는 실제 사고 발생을 단정하지 않습니다.
- 모든 결과는 시뮬레이션 기반 위험도 예측입니다.
- 최종 운영 판단은 관리자와 기사 확인을 거쳐야 합니다.
- 실제 고객 주소, 전화번호, GPS, 민감 개인정보는 사용하지 않습니다.
