# SafeRoute AI 데모 스크립트

## 1. 문제 제기

빠른 배송만으로는 현장의 안전을 설명할 수 없습니다. 같은 배송량이라도 비, 야간, 언덕, 좁은 골목, 주정차 난이도, 긴 근무 시간이 겹치면 사고 위험은 크게 높아집니다.

SafeRoute AI는 기사 감시 시스템이 아닙니다. 기사와 관리자가 사고를 예방하기 위해 현재 위험을 이해하고, 경로와 물량과 일정을 더 안전하게 조정하도록 돕는 의사결정 보조 시스템입니다.

## 2. 핵심 메시지

> 우리는 배송을 더 빠르게 만드는 것이 아니라, 더 안전하게 만드는 AI를 만듭니다.

## 3. 3분 시연 흐름

1. 데모 시나리오에서 `관악구 우천 고위험 시나리오`를 선택합니다.
2. 상단의 안전 의사결정 요약에서 세 가지 질문에 답합니다.
   - 누가 위험한가?
   - 왜 위험한가?
   - 지금 무엇을 해야 하는가?
3. 기사 위험도 목록에서 `지원 필요 기사`를 선택합니다.
4. 상세 패널에서 현재 사고 위험도와 남은 배송, 누적 근무, 날씨 조건을 확인합니다.
5. 위험 원인 TOP 5에서 남은 배송, 누적 근무, 피로, 우천, 언덕/빌라 밀집 같은 원인을 설명합니다.
6. 최단 경로와 안전 경로를 비교합니다.
7. 시간이 조금 더 걸리더라도 위험도가 낮아지는 안전 경로를 AI가 권장한다는 점을 강조합니다.
8. AI 권장 조치 목록에서 물량 분산, 휴식, 배송 순서 조정, 저우선순위 지연 안내를 확인합니다.
9. `권장안 시뮬레이션 적용` 버튼을 누릅니다.
10. 추천 적용 전/후 영향과 관리자 영향 요약에서 위험도 감소, 분산 배송 수, 지연 안내 수, 휴식 권장 여부를 확인합니다.

## 4. 발표 문장 예시

SafeRoute AI는 배송 속도를 최적화하는 시스템이 아닙니다. 현재 배송 상황에서 어떤 기사에게 안전 지원이 필요한지 예측하고, 왜 위험한지 설명하며, 경로와 일정을 어떻게 바꾸면 위험을 줄일 수 있는지 제안합니다.

이 데이터는 실제 개인정보가 아닌 시뮬레이션 데이터입니다. 하지만 배송량, 피로, 날씨, 지역 특성, 경로 위험을 구조화했기 때문에 실제 서비스로 확장 가능한 의사결정 흐름을 보여줄 수 있습니다.

## 5. 책임 원칙

- AI는 의사결정을 보조합니다.
- 최종 운영 판단은 현장 관리자와 기사에게 있습니다.
- 추천 적용 결과는 시뮬레이션입니다.
- 실제 고객 주소, 전화번호, GPS, 민감 개인정보는 사용하지 않습니다.
- LLM은 위험 점수를 계산하지 않습니다. 위험 점수는 Risk Engine이 계산합니다.
